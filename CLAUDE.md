## Overview

This is a SoundCloud automation tool that fetches new tracks and albums from followed artists and manages them in playlists. The system runs weekly, capturing content from the past 7 days.

## Key Files and Their Roles

- **`runzIt.py`**: Main entry point that orchestrates the entire workflow
  - Calculates the time window (past 7 days from midnight Eastern Time)
  - Calls both songs and albums modules

- **`getNewSongs.py`**: Fetches and manages individual tracks
  - Paginated fetch from `/me/activities/tracks`
  - Filters for original uploads (not reposts)
  - Supports three filter modes: `activity`, `upload`, or `both` (controlled by `SC_FILTER_MODE` env var)
  - Updates playlist `1907305003` by default (override with `SC_PLAYLIST_ID` env var)
  - **IMPORTANT**: Uses replace-all strategy (clears playlist and sets to only new tracks)

- **`getNewAlbums.py`**: Fetches and likes albums/EPs/compilations
  - Fetches from `/me/activities/all/own`
  - Only processes playlists with type in `{"ALBUM", "EP", "COMPILATION"}` and >2 tracks
  - Automatically likes albums using `SCClient.likeIt()`
  - Tracks liked albums in `albums_added.txt` to avoid duplicates
  - **Hardcoded path**: Uses `/Users/WillyTardif/Documents/SC_App/albums_added.txt` for reading existing albums (line 135)

- **`SC_Token.py`**: One-time OAuth token generation script
  - Exchanges authorization code for access/refresh tokens
  - Requires `SC_CLIENT_ID` and `SC_CLIENT_SECRET` from `secrets.env`
  - Outputs token data that should be saved to `sc_token.json`

- **`soundAuthen` module**: External dependency (not in repo)
  - Provides `SCClient` class for authenticated SoundCloud API calls
  - Must be available in Python path when running
  - Handles token refresh and API requests

- **`sc_token.json`**: Stores OAuth tokens
  - Contains `access_token`, `refresh_token`, `expires_at`, etc.
  - Generated by `SC_Token.py` or token refresh flow

- **`secrets.env`**: Environment variables (not in version control)
  - `SC_CLIENT_ID`: SoundCloud OAuth client ID
  - `SC_CLIENT_SECRET`: SoundCloud OAuth client secret

## Running the System

```bash
# Run the full weekly update (songs + albums)
python3 runzIt.py

# Run songs only
python3 getNewSongs.py

# Run albums only
python3 getNewAlbums.py

# Generate new OAuth tokens (when needed)
python3 SC_Token.py
```

## Architecture Notes

### Time Filtering Strategy

The songs module supports flexible time filtering via `SC_FILTER_MODE`:
- **`activity`** (default): Filter by when the track appeared in your feed (`activity.created_at`)
- **`upload`**: Filter by actual upload time (`origin.created_at`)
- **`both`**: Require both timestamps to be within the window

The cutoff calculation in `runzIt.py` uses Eastern Time (America/New_York) and starts from midnight 7 days ago.

### Pagination and Early-Stop

Both modules implement smart pagination:
1. Follow `next_href` links from SoundCloud's linked partitioning API
2. Early-stop when newest activity on a page is older than cutoff (activities are reverse-chronological)
3. Songs module also checks `future_href` for real-time updates

### API Access Parameter

All API calls include `access=playable,preview,blocked` to ensure visibility of all track states. This is enforced via `_ensure_access_in_url()` helper.

### Deduplication

- **Songs**: Deduplicates by `track_id` after fetching
- **Albums**: Tracks all liked albums in `albums_added.txt` to prevent re-liking

### Playlist Update Strategy

Songs module uses **replace-all** updates (`preserve_existing=False`):
- Playlist is cleared and set to exactly the new tracks
- Does NOT preserve existing tracks
- Sends URNs in format `soundcloud:tracks:{id}` via PUT to `/playlists/{id}`

## Dependencies

The codebase requires:
- `requests`: HTTP client for SoundCloud API
- `python-dotenv`: Environment variable loading
- `alive_progress`: Progress bars during pagination
- `soundAuthen`: External module providing `SCClient` (must be in Python path)

1. **Path Hardcoding**: `getNewAlbums.py:135` has absolute path `/Users/WillyTardif/Documents/SC_App/albums_added.txt` that differs from the repo location
2. **Missing Module**: `soundAuthen` is imported but not present in the repository
3. **Token Expiry**: Tokens expire after ~1 hour; refresh logic must be in `SCClient`
4. **No Error Recovery**: Failed API calls will raise exceptions without retry logic
